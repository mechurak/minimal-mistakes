GitHub를 통해 공동 작업을 하려면
Collaborator 로 등록해서 git push 권한을 줘서 같은 repo 에서 작업하거나,  

권한을 주거나 받을 수 있는 상황이 아니면, fork를 통해 작업하고 PR를 할 수 있다.

두 경우 모두 PR이 유용해 보인다. 임시 브랜치를 새로 따서 작업 한 다음 푸쉬해서 PR를 하고, 리뷰 후 머지되면 임시 브랜치는 삭제한다.
